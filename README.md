## Description

A simple end-to-end MLOps pipeline built on the classic Titanic survival dataset. 

This project demonstrates a full machine learning workflow emphasizing MLOps best practices, including:
- **Data Ingestion**: Automated downloading of the Titanic dataset using `kagglehub`.
- **Data Preprocessing**: Modularized cleaning and feature engineering pipelines.
- **Model Training**: Comparing multiple models (e.g., Logistic Regression, Random Forest).
- **Evaluation & Metrics**: Tracking and saving metrics in machine-readable (`.json`) formats.
- **Configuration Management**: Managing hyperparameters and pipeline settings using `Hydra`.
- **Reproducibility**: Environment management using `uv` and modular architecture.

---

## Prerequisites

- **Python**: `3.11`
- **Package Manager**: `uv` (Recommended for fast dependency resolution) or `pip`
- **Make**: Available for running Makefile convenience commands (optional but recommended)

---

## Installation & Setup

1. **Clone the repository** (if you haven't already):
   ```bash
   git clone <your-repository-url>
   cd Titanic_project_MLOps_lab_0
   ```

2. **Create the virtual environment**:
   You can use the provided Makefile command to create a `uv` virtual environment:
   ```bash
   make create_environment
   ```
   *Activate the environment:*
   - Windows: `.\.venv\Scripts\activate`
   - Unix/macOS: `source ./.venv/bin/activate`

3. **Install Dependencies**:
   Sync all required packages using `uv`:
   ```bash
   make requirements
   ```
   *(Alternatively, if not using Make: `uv sync`)*

4. **Kaggle API Credentials**:
   To download the data automatically, ensure you have your `kaggle.json` API token placed either in the project root or your global Kaggle config folder (`~/.kaggle/`).

## Usage

### Running the Full Pipeline

The primary entry point orchestrates downloading data, preprocessing, training, and evaluation. To execute the entire pipeline end-to-end:

```bash
python -m src.trainer
```

### Configuration (Hydra)

This project uses [Hydra](https://hydra.cc/) for hierarchical configuration. The default settings are managed in `conf/config.yaml`. 

You can easily swap out models or change parameters directly from the command line without modifying the code. For example, to switch from the default model to a Random Forest:

```bash
python -m src.trainer model=random_forest
```
